# 云效 Bug 修复技能 - 快速开始

## 📦 5 分钟上手

### 第 1 步：确认前置条件

你需要：
1. ✅ 一个支持 Agent Skills 的 AI Agent（Claude Code、VS Code + Claude、Cursor 等）
2. ✅ 云效 MCP 服务器已配置
3. ✅ 云效账号和项目访问权限

### 第 2 步：安装技能

**下载到你的技能目录：**
```bash
cd ~/.claude/skills/  # 或你的 Agent 技能目录
git clone https://github.com/willnie9/yunxiao-bug-fix.git
```

或手动复制整个文件夹到技能目录。

### 第 3 步：配置云效 MCP

在你的 MCP 配置文件中添加：

```json
{
  "mcpServers": {
    "mcp_aliyun_yunxiao": {
      "command": "node",
      "args": ["/path/to/yunxiao-mcp-server/index.js"],
      "env": {
        "YUNXIAO_ACCESS_TOKEN": "你的云效访问令牌"
      }
    }
  }
}
```

**获取访问令牌：**
1. 登录云效：https://devops.aliyun.com
2. 进入个人设置 → 访问令牌
3. 创建新令牌并复制

### 第 4 步：验证安装

重启你的 Agent，然后问它：

```
列出我的可用技能
```

如果看到 `yunxiao-bug-fix` 就说明安装成功了！

## 💡 第一次使用

### 场景 1：修复一个具体的 Bug

**你说：**
```
帮我修复这个 bug：https://devops.aliyun.com/projex/project/xxx/bug/TXRP-592
```

**Agent 会：**
1. 📥 获取 Bug 详情并展示给你确认
2. ⏸️ 等你回复"确认"或"开始修复"
3. 📎 下载截图和附件，查看完整上下文
4. 🔍 定位相关代码文件
5. 🔧 分析问题并提出修复方案
6. ⏸️ 等你确认修复方案
7. ✅ 修复代码后再次等你确认
8. 📝 更新云效状态并添加详细评论

**关键点：每一步都会等你确认，不会擅自操作！**

### 场景 2：查看你的待办 Bug

**你说：**
```
显示我的待处理 bug
```

**Agent 会：**
1. 🔍 搜索分配给你的、未关闭的 Bug
2. 📋 展示列表，包含标题、优先级、状态
3. ⏸️ 等你选择要处理哪一个

**然后你可以说：**
```
修复 TXRP-592
```

### 场景 3：搜索特定 Bug

**你说：**
```
查找与"登录"相关的 bug
```

或者：

```
有哪些高优先级的 bug？
```

或者：

```
最近 3 天创建的 bug
```

## 🎯 核心特性

### ✅ 用户确认工作流

技能**绝不**会在未经你确认的情况下：
- 修改代码
- 更新云效状态
- 添加评论

每一步都会：
1. 📊 展示将要做什么
2. ⏸️ 等待你的确认
3. ✅ 得到确认后才执行

### 🔄 智能容错

- 状态 ID 错误？自动查询正确的 ID 并重试
- 附件下载失败？优雅处理并提示
- 网络超时？自动重试

### 📋 完整的上下文收集

修复 Bug 前，技能会：
- ✅ 获取 Bug 详情
- ✅ 读取所有评论（可能有重要信息）
- ✅ 下载所有截图（必须查看，不能跳过）
- ✅ 检查附件列表

### 📝 结构化评论

修复完成后，会自动生成格式化的评论：

```markdown
✅ 已修复 | ⏱️ 修复时间：2026-04-28 14:30
❗ 感谢测试同学 @张三 精准报 Bug！
⭐ 测试人，YYDS！

⬇️ 验证步骤
1. 打开登录页面
2. 点击登录按钮
3. 应该能正常登录

⚠️ 问题原因
登录按钮的 z-index 设置过低，被遮罩层挡住了

⚡ 修复方案
- 将登录按钮 z-index 从 1 提升到 100
- 给遮罩层添加 pointer-events: none

➡️ 涉及文件
`src/components/LoginButton.vue`
```

## 🔧 常见问题

### Q: 技能没有出现在列表中？

**A: 检查以下几点：**
1. 文件夹是否在正确的技能目录？
2. 文件夹名是否为 `yunxiao-bug-fix`？
3. `SKILL.md` 文件是否存在？
4. 是否重启了 Agent？

### Q: 提示找不到云效 MCP 工具？

**A: 检查 MCP 配置：**
1. MCP 配置文件路径是否正确？
2. 云效 MCP 服务器是否正确配置？
3. 访问令牌是否有效？
4. 尝试重启 Agent

### Q: 下载截图失败？

**A: 可能的原因：**
1. 网络连接问题
2. 云效访问令牌过期
3. 附件链接已失效

**解决方法：**
- 检查网络连接
- 更新访问令牌
- 手动在云效中查看截图

### Q: 状态更新失败？

**A: 技能会自动处理：**
1. 自动查询项目的实际状态流
2. 找到正确的状态 ID
3. 自动重试更新

如果还是失败，可能是：
- 没有权限更新该 Bug
- 状态转换不被允许

### Q: 可以自定义评论模板吗？

**A: 可以！**
编辑 `assets/评论模板.md` 文件，修改默认模板。
