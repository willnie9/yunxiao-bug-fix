# 默认评论模板

本模板用于在修复 Bug 后向云效工作项添加评论。

## 模板变量

- `{fix_time}` - 修复完成时间戳（YYYY-MM-DD HH:mm）
- `{verifier_name}` - 验证者姓名（从工作项获取）
- `{verification_steps}` - 验证步骤的编号列表
- `{problem_cause}` - 导致 Bug 的详细原因说明
- `{fix_plan}` - 修复更改的要点列表
- `{affected_files}` - 修改的文件列表（用反引号包裹）

## 默认模板

```markdown
✅ 已修复 | ⏱️ 修复时间：{fix_time}
❗ 感谢测试同学 @{verifier_name} 精准报 Bug，定位准确，帮助快速修复！
⭐ 测试人，YYDS！每一个被你点亮的缺陷，都是产品进步的台阶。

**⬇️ 验证步骤**
{verification_steps}

**⚠️ 问题原因**
{problem_cause}

**⚡ 修复方案**
{fix_plan}

**➡️ 涉及文件**
{affected_files}
```

## 支持的 Emoji

云效评论中支持以下 emoji：

- ✅ ❌ ⚠️ ✔️ ✖️
- ➡️ ⬅️ ⬆️ ⬇️
- ℹ️ ❗ ❓
- ⭐ ⚡ ♻️ ⏱️
- 🐛 🔧 🎯 📝 📦 🔍

## Agent 中的使用

Agent 会自动：
1. 需要时读取此模板文件
2. 用实际值替换变量
3. 正确格式化列表和代码块
4. 将评论添加到云效工作项

## 最佳实践

### 验证步骤
- 具体且可操作
- 包含预期结果
- 提及任何特殊条件（浏览器、设备等）
- 为便于跟随而编号步骤

### 问题原因
- 解释技术根本原因
- 包含相关的代码逻辑
- 提及任何边界情况或条件
- 帮助未来的开发者理解问题

### 修复方案
- 列出每个更改
- 解释为什么每个更改修复了问题
- 提及任何权衡或考虑
- 如果相关，包含测试方法

### 涉及文件
- 使用从项目根目录的相对路径
- 用反引号包裹以提高可读性
- 按重要性顺序列出
- 如果适用，包含源文件和测试文件

## 模板选择

要使用自定义模板，修改此文件或创建新的模板文件并在 Agent 配置中引用它。

Agent 将使用此文件中找到的第一个模板块（三个反引号之间）作为默认模板。

## 使用示例

**输入变量：**
```javascript
{
  fix_time: "2026-04-28 14:30",
  verifier_name: "张三",
  verification_steps: "1. 打开登录页面\n2. 点击登录按钮\n3. 应该能正常登录",
  problem_cause: "登录按钮的 z-index 设置过低，被遮罩层挡住了",
  fix_plan: "- 将登录按钮 z-index 从 1 提升到 100\n- 给遮罩层添加 pointer-events: none",
  affected_files: "`src/components/LoginButton.vue`"
}
```

**生成的评论：**
```markdown
✅ 已修复 | ⏱️ 修复时间：2026-04-28 14:30
❗ 感谢测试同学 @张三 精准报 Bug，定位准确，帮助快速修复！
⭐ 测试人，YYDS！每一个被你点亮的缺陷，都是产品进步的台阶。

**⬇️ 验证步骤**
1. 打开登录页面
2. 点击登录按钮
3. 应该能正常登录

**⚠️ 问题原因**
登录按钮的 z-index 设置过低，被遮罩层挡住了

**⚡ 修复方案**
- 将登录按钮 z-index 从 1 提升到 100
- 给遮罩层添加 pointer-events: none

**➡️ 涉及文件**
`src/components/LoginButton.vue`
```
